Экзаменационное задание

В качестве экзаменационного задания предлагаем Вам создать небольшую распределенную систему для учета биржевой активности. Система состоит из четырех частей: база данных, рабочее место брокера, WEB-приложение, EJB-компоненты.

Функциональные возможности системы.

Рабочее место брокера

Брокер вносит информацию о факте продаже ценной бумаги (Акции), указывается сокращенное название ценной бумаги (к примеру акции Microsoft обозначаются как MSFT), кол-во ценных бумаг которые были проданы и цена за ценную бумагу. Это информация с помощью EJB-компонента сохраняется в базу данных.

Каждый брокер до начала работы должен аутентифицироватся. Пользовательский интерфейс брокера должен отображать записи созданые другими брокерами (для этого можно использовать таймер (класс Timer) который по истичению определенного интервала времени, будет вызывать метод EJB компонента и получать новые записи. Обратите внимание на то что удаленный метод может выполняться достаточно долго, следовательно, рациональное решение - это выполнить метод обновления данных асинхронно (в отдельном потоке)).

WEB приложение

WEB приложение отображает последнии сделки на бирже в виде таблицы (Акции, Кол-во, Цена, Брокер). Каждая строка таблицы содержит ссылку "Дополнительно", по нажатию на которую отображаются все сделки по данной ценной буумаге за сегодня.
